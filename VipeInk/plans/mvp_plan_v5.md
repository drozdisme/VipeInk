## Primary execution logic
Build one reusable, cheap demo core first.
That core must be usable for:
- product demo
- grant applications
- competitions
- investor / partner conversations

## 2-week objective
- define and build smallest convincing TON settlement / L2 abstraction demo
- expose API + state flow + record layer
- make it presentable

## 4-week objective
- polish demo
- prepare application assets for grants / competitions
- adapt story into 2-3 narratives without rebuilding core

## Selection logic
Keep the main path unless another path proves clearly better on:
- speed
- cost
- access to capital
- access to users / partners
- strategic leverage

## Key decision rule
Do not chase multiple product builds.
Chase one core build and multiple opportunity wrappers.
