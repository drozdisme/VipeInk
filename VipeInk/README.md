# VipeInk planning vault

## Структура
- `Граф развития Vipeink.md` - центральный стратегический граф
- `MVP.md` - центральный хаб reusable core MVP
- `План развития.md` - выбор основного пути и альтернатив
- `Следующие шаги.md` - короткий исполнимый фокус
- `plans/` - версионируемые планы MVP
- `strategy/` - основной путь и независимые альтернативы
- `mvp/` - продуктовый слой MVP
- `mvp/features/` - ключевые capability notes
- `mvp/phases/` - этапы сборки и проверки
- `research/` - исследования и внешние opportunity maps
- `reference/` - исходные материалы и long-term reference
- `reference/hubs/` - хабы для исходников, ICP и future architecture
- `archive/` - второстепенные или сырые заметки, не подключенные к главному графу

## Правило чистоты
- В корне держать только главные хабы и README.
- Всё, что не является главным входом в граф, должно жить в тематической папке.
- Один reusable core, много упаковок поверх него.
