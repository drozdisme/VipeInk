## Центральная логика
У Vipeink один основной путь развития и несколько независимых альтернативных путей усиления.

Главное правило:
- не строить несколько разных продуктов
- строить один reusable core
- использовать его в разных нарративах и каналах роста

## Порядок следования
1. Собрать reusable core MVP
2. Упаковать его как TON settlement / L2 abstraction demo
3. Поверх него запускать grant / contest / ecosystem routes
4. Если один из альтернативных путей даёт более быстрый и дешёвый рычаг, усиливать его без смены core

## Ключевая проверка любого следующего шага
Любой шаг должен отвечать минимум на один из вопросов:
- ускоряет ли это запуск
- снижает ли это стоимость
- даёт ли это funding / credits / visibility / partnerships
- усиливает ли это reusable core

## Что НЕ делаем
- не уходим в полный rollup build
- не распадаемся на consumer app, bank product и infra product одновременно
- не делаем отдельный стек под каждую opportunity
